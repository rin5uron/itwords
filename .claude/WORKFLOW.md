# 🛠️ 日々の作業フロー

このファイルは、Claude Codeでの日々の作業の進め方を記録しています。

---

## 📝 1日の作業記録の書き方

### ルール
- **作業終了時**に必ず`docs/process-log.md`に記録すること
- **日付別にセクションを追加**（例: `## 2026-01-15: タイトル`）
- **実施内容・背景・成果**を明記

### テンプレート
```markdown
## YYYY-MM-DD: [タイトル]

### 実施内容

#### 1. [作業項目1]
**背景**:
- なぜこの作業をしたのか

**実装**:
- 何をしたのか（Before/After）

**コミット**: `コミットメッセージ`

---

### 成果
- 数値で測れる成果
- 期待効果
```

### 記録すべきファイル
- **`docs/process-log.md`** - 日々の作業ログ（時系列）
- **`docs/all-pages-seo-strategy.md`** - SEO戦略一覧

---

## 🔍 SEO調査の記録方法

### ルール
- **Search Consoleでデータ取得時**に必ず`docs/all-pages-seo-strategy.md`を更新
- **競合調査した時**も同ファイルに追記
- **勝算パーセントを必ず記入**（根拠を持って）

### 更新すべき内容
1. **ページ一覧表を更新**
   - 新規ページを追加
   - 勝算が変わったページを更新
   - 最終更新日を記入

2. **成果サマリーを追記**
   ```markdown
   ### 📊 YYYY-MM-DDの成果
   **新規追加**: Xページ
   **強化完了**: Yページ
   **1位獲得見込み高いページ（🟢）**: Z → W に増加
   ```

### 記録すべき項目
| 項目 | 説明 |
|------|------|
| ページ | `/terms/xxx` |
| メインキーワード | 「〇〇 とは」 |
| 1位獲得用ニッチキーワード | 「〇〇 とは デモ付き」 |
| 競合強度 | 🟢少ない / 🟡中程度 / 🔴多い |
| 勝算 | 60-98% |
| 状態 | ✅完了 / 🔄改善中 / 📋計画 |
| 最終更新 | YYYY-MM-DD |

## 📊 振り返り時に見るべきファイル

### 日々の作業を振り返る
👉 **`docs/process-log.md`**
- 何をいつやったか時系列で確認
- 過去の実装方法を参照
- 同じミスを繰り返さないため

### SEO状況を確認する
👉 **`docs/all-pages-seo-strategy.md`**
- 全ページのSEO状況を一覧確認
- どのページが1位を狙えるか把握
- 次に強化すべきページを判断

### キーワード戦略を確認する
👉 **`docs/seo-keyword-strategy.md`**
- キーワード選定の基準
- 競合分析の結果
- ロングテール戦略

## ✅ 作業終了時のチェックリスト

毎日の作業終了時に確認：

- [ ] `docs/process-log.md`に今日の作業を記録した
- [ ] 新規ページ追加 or SEO改善した場合、`docs/all-pages-seo-strategy.md`を更新した
- [ ] コミットメッセージに作業内容を明記した
- [ ] デプロイが成功したか確認した

---

## 🎯 次回作業時の流れ

1. **前回の作業を振り返る**
   - `docs/process-log.md`で前回何をしたか確認
   - `docs/all-pages-seo-strategy.md`で次に強化すべきページを確認

2. **Search Consoleでデータ確認**
   - アクセス数の変化
   - 新しい検索クエリ
   - クリック率が低いページ

3. **優先順位を決めて作業開始**
   - 🔴アクセス多いが改善余地あり → 優先度高
   - 🟢新規ページ追加でニッチ狙い → 優先度中
   - 🟡既存ページ強化 → 優先度低

4. **作業終了後、必ずログ記録**
   - `docs/process-log.md`
   - `docs/all-pages-seo-strategy.md`
