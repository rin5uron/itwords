# SEO title/description チェック・修正

Search Consoleのキーワードデータを受け取り、各ページのtitle・descriptionに検索キーワードが合っているか確認・修正する。

## 使い方

コンソールのキーワード・表示回数・クリック数データを貼り付けて実行する。

```
/seo-check
キーワード　クリック　表示
ssh         0        4
https       0        4
...
```

## 実行前に読むファイル

不要（データを見て動く）

## 実行手順

1. ユーザーが貼り付けたキーワード・表示・クリックデータを整理する
2. 「表示が多くてクリック0」のキーワードを優先とする（損失が大きい順）
3. 各キーワードに対応するページのmetadataを収集する
   - layout.tsx がある場合はそちらを優先
   - ない場合は page.tsx の metadata を見る
4. 以下の視点で「直すべき」かどうかを判断する：
   - キーワードの読み方がtitle/descriptionに無い（例：「エイチティーティーピーエス」「ヴァーセル」）
   - 検索者の意図と離れているtitle（例：「小学生でもわかる」だけで検索キーワードが無い）
   - 英語キーワードで検索されているのに英語表記がdescriptionに無い
   - titleが30日本語文字を超えて途切れている可能性がある
   - descriptionが短すぎる（60文字未満）
5. 修正すべきページを優先度順にリストアップし、実装する
6. 修正時のポイント：
   - titleは約30日本語文字以内に収める
   - descriptionは80〜120日本語文字が理想
   - 読み方キーワードはdescriptionに「（カタカナ）」で入れる
   - 英語キーワードの場合は「英語（日本語）とは？」の形で入れる
   - openGraph・twitter のtitle/descriptionも同様に更新する
7. `npx next build` で確認
8. 修正内容をまとめて報告する

## 判断の優先度

| 優先度 | 条件 |
|--------|------|
| 最高 | 表示100回以上・クリック0 |
| 高 | 表示50回以上・クリック0 |
| 中 | 表示10回以上・クリック0 |
| 低 | クリック>0だが読み方キーワードが無い |

## 注意

- dateModified は更新しない（メタデータのみの変更なので）
- keywords 配列には検索キーワードに合わせて追加してもよい
- トップページのmetadataは「IT用語」のような強い競合キーワードなので、表示が少なくても変更しにくい場合がある
