# 既存ページ改善

既存の用語ページのコンテンツ内容・テンプレート準拠・デモの追加・構成の見直しを行う。

## 使い方

```
/improve-page [ページ名]
```

例：`/improve-page cpu`

## 実行前に読むファイル（必須）

1. `.claude/TEMPLATE_RULES.md` - テンプレート準拠ルール
2. `docs/templates/page-template.tsx` - ページテンプレート
3. `.claude/UPDATE_DATE_POLICY.md` - 更新日ポリシー
4. 対象ページ本体（`app/terms/[ページ名]/page.tsx`）
5. 対象ページの layout.tsx（あれば）

## 実行手順

1. 対象ページを読む
2. テンプレート準拠チェックを行う（以下の項目）
3. ユーザーの要求に基づいてコンテンツを改善する
4. 改善後にテンプレート準拠を再確認する
5. `npx next build` で確認
6. コンテンツの実質的な変更があった場合は dateModified を今日の日付にする

## テンプレート準拠チェック

以下に該当する場合は修正する：

- [ ] 「概要」セクションが最初にある（無い場合は追加）
- [ ] TermPageHeader を使っている（個別コンポーネントでない）
- [ ] セクション順序が「概要 → 実例 → デモ → 補足 → 関連用語 → FAQ」になっている
- [ ] `className="term-comparison"` と `className="comparison-table"` がある
- [ ] StructuredData（FAQPage + Article）がある
- [ ] summaryItems がページの構成と合っている
- [ ] インライン目次・ヘッダースタイルを書いていない

## コンテンツ改善のポイント

- デモの追加・改善の場合は、既存デモと重複しないように配置する
- 説明の流れが「わかりやすい順」になっているか確認する
- 身近な例や具体例が入っているか確認する
- 関連用語に抜けがないか確認する
- FAQが現在のコンテンツと合っているか確認する
