## 🎯 コンテンツ戦略

### SEOキーワード戦略
以下の基準で用語を選定：
1. **基本情報**: IT初心者が頻繁に検索する用語
2. **今後の必要性**: 自分の学習・開発に役立つ
3. **SEO競合性**: 競合が少なく上位表示が狙える
4. **検索ボリューム**: 月間検索数がある程度ある

### 優先して追加する用語（7-10ページ）

#### 🟢 最優先（基本中の基本）
1. **HTML** - Web開発の基礎、検索数多、競合は多いがロングテール狙い
2. **CSS** - HTMLとセット、同上
3. **JavaScript** - フロントエンド必須、同上
4. **HTTP/HTTPS** - Web通信の基本、セキュリティ関連で需要
5. **Cookie** - プライバシーポリシーとも関連、初心者の疑問多

#### 🟡 次点（差別化しやすい）
6. **Vercel** - 最新のホスティング、競合少なめ
7. **GitHub** - 開発者必須ツール、初心者向け解説は少ない
8. **npm** - パッケージ管理、初心者がつまずきやすい
9. **環境変数** - 実践的、セキュリティ関連
10. **デプロイ** - Web開発の最終工程、初心者の疑問多

## 📋 用語ページの構成（テンプレート）

各用語ページに含める要素:
1. **タイトル**: 用語名
2. **読み方**: ふりがな
3. **カテゴリ**: Web技術/開発ツール/セキュリティなど
4. **難易度**: 初級/中級/上級

6. **詳細説明**: 3-5段落
7. **実例・デモ**: 可能な限り実際に動くもの
8. **関連用語**: 2-3個リンク
9. **メタ情報**: 作成日、更新日

## 🔄 2026年1月17日 追加改善計画

### 🚨 審査落ちの原因分析

アドセンス審査で「有用性の低いコンテンツ」と評価された主な理由:

1. **コンテンツのボリューム不足**（最大の問題）
   - 薄いページが多い（3,000文字未満: 9ページ / 全体の30%）
   - 最優先改善: 5ページ（2,500〜2,900文字）
   - 改善推奨: 4ページ（3,100〜3,600文字）

2. **トップページの説明が少ない**
   - 現在約200文字 → 800〜1,000文字に拡充が必要

3. **サイト運営期間が短い**
   - 制定日: 2026年1月6日（運営期間: 約1〜2週間）
   - 推奨: 最低1ヶ月以上の運営実績

4. **更新履歴・更新日が不明**
   - 各ページに「最終更新日」の表示がない
   - 活発なサイトであることをGoogleに示せていない

### 📋 緊急対応タスク

#### 🔴 最優先（今すぐ実施）

1. **トップページ改善**
   - 「制作の背景と意図」セクションを拡充
   - 現在約200文字 → 目標800〜1,000文字
   - サイトの詳しい説明、対象読者、他の辞典との違い、学習の進め方を追加

2. **薄いページの改善（5ページ）**
   - `/terms/queue/page.tsx`（2,500文字 → 3,500文字以上）
   - `/terms/localstorage/page.tsx`（2,700文字 → 3,500文字以上）
   - `/terms/zero-day/page.tsx`（2,700文字 → 3,500文字以上）
   - `/terms/json/page.tsx`（2,800文字 → 3,500文字以上）
   - `/terms/exploit/page.tsx`（2,900文字 → 3,500文字以上）

3. **最終更新日の管理**
   - **表示ルール**: GitHub/Claudeなど情報の鮮度に左右される技術のみ画面に表示
   - **基本的な用語**: コード上で管理（メタデータまたはコメント）
   - **理由**: 普遍的な用語に日付表示は不自然、UX優先

#### 🟡 次に重要（1週間以内）

4. **改善推奨ページの改善（4ページ）**
   - `/terms/html/page.tsx`（3,100文字 → 4,000文字以上）
   - `/terms/cve/page.tsx`（3,200文字 → 4,000文字以上）
   - `/terms/white-hat-hacker/page.tsx`（3,600文字 → 4,000文字以上）
   - `/terms/stack/page.tsx`（3,600文字 → 4,000文字以上）

5. **継続的な更新**
   - 週1〜2回のペースで既存ページを更新
   - 更新履歴を作り、活発なサイトであることを示す

### 📝 コンテンツ拡充方針（読みやすさ重視）

**基本方針**: ボリュームを増やしつつ、読みやすさを維持

#### ✅ 推奨する追加要素（+1,000文字の内訳例）

1. **実例を3〜5個追加**（各200〜300文字）
   - 日常で見かける実例
   - 「こういう時に使われている」という具体例
   - 視覚的に理解しやすい内容

2. **FAQ形式で疑問に答える**（各100〜200文字）
   - 「よくある質問」セクション
   - 短い質問と回答のセット（5つ以上）
   - 読み飛ばしやすい構造

3. **体験デモ・インタラクティブ要素の説明拡充**（+300文字）
   - デモの使い方ステップを詳しく
   - 「何がわかるか」を明確に説明
   - デモ結果の見方を解説

4. **比較表やリスト**（+200文字）
   - 類似用語との違い
   - 視覚的で読みやすい
   - スキャンしやすい構造

#### ❌ 避けるべき追加方法

- 長い文章をダラダラ追加
- 難しい技術用語の羅列
- 関係ない情報の追加
