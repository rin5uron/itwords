# issue 作成ルール

## 作成方法

- issue は **CLI 経由**で作成する。`gh issue create --title "..." --body "..."` を使う。
- GitHub の Web から新規作成するのではなく、リポジトリ内で `gh` コマンドで作成する。

---

## タイトル

- **先頭に【】で種別を入れる。** 例：【課題】【学び】【ログ】【方針】。
- [ロードマップ][議事録] のように **[]** でも可。
- **日付はタイトルに入れない。** 日付は **本文の最初** に1行で書く。**右寄せ**で `2026-02-18` の表記だけ（「日付:」などは付けない）。右寄せは `<p align="right">2026-02-18</p>` のように書く。
- **長さの目安:** できるだけ短く。議事録は「[議事録] テーマ（キーワードのみ）」にし、詳細は本文に。目安 50 字以内。
- **【学び】で一緒に学んだこと:** タイトルには学んだテーマのワードを並べておく。例：【学び】Cloudflare・被リンク・API・タイムスリープ

---

## 種別とラベル（タグ）

### 基本ラベル（種別用）

| 種別 | ラベル | 意味 |
|------|--------|------|
| 【課題】 | なし（作業内容ラベルのみ） | やること・ToDo。実施記録はその issue の本文やコメントに書く。 |
| 【方針】 | `policy` | ルール・決めごと（「こうする」と決めたこと）。 |
| 【ログ】 | `log` | 記録用の issue。やったこと・起きたことを残すだけ。やるタスクはない。 |
| 【学び】 | `study` | 学んだことのまとめ。 |
| [ロードマップ] | `roadmap` | フェーズやマイルストーンの計画。 |
| [議事録] | `meeting` | 打ち合わせの記録。 |

### 作業内容ラベル（課題用）

【課題】には作業内容に応じて以下のラベルを付ける：

| ラベル | 用途 | 色 |
|--------|------|-----|
| `SEO` | 検索エンジン最適化（title/description/キーワード/構造化データ） | #0e8a16 |
| `description` | title・descriptionのメタ情報 | #5319e7 |
| `コンテンツ` | 記事の執筆・品質管理 | #d876e3 |
| `UI/UX` | デザイン・使いやすさ | #c5def5 |
| `next` | Next.js実装タスク | #a2eeef |
| `仕組み作り` | ワークフロー・ルール・テンプレート整備 | #7057ff |
| `bug` | 不具合 | #d73a4a |
| `documentation` | ドキュメント改善・追加 | #0075ca |

### ラベルの付け方

- **【課題】**: 作業内容ラベル1つ（例: `SEO`のみ）
- **【ログ】**: `log`ラベルのみ
- **【学び】**: `study`ラベルのみ
- **複数ラベル可**: SEO+bug など、内容が複合する場合のみ

### その他のルール

- **作業した人・担当者は assignees（担当者）で表す。** ラベルではなく GitHub の Assignees に自分や一緒にやった人を追加する。例: `gh issue edit <番号> --add-assignee @rin5uron`
- **お互いの名前はアカウント名で呼ぶ。** issue・コメント・ドキュメント内では @rin5uron のように GitHub アカウント名を使う。

**作成例（ラベル・assignee 付き）**

```bash
# 【課題】の例（SEO改善）
gh issue create --title "【課題】画像alt属性の追加" --body "<p align=\"right\">2026-03-08</p>\n\n（本文）" --label "SEO" --assignee "@me"

# 【ログ】の例
gh issue create --title "【ログ】SEO監査実施" --body "<p align=\"right\">2026-03-08</p>\n\n（本文）" --label "log" --assignee "@me"

# 【学び】の例
gh issue create --title "【学び】構造化データ・Core Web Vitals" --body "<p align=\"right\">2026-03-08</p>\n\n（本文）" --label "study" --assignee "@me"
```

---

## 「上書きして」のとき

- **「上書きして」と言われたときは issue の本文（body）を編集する。** コメント欄には書かない。

---

## 振り返り

- タイトルとラベルの対応は .cursorrules の「issue の作成」にも記載している。
