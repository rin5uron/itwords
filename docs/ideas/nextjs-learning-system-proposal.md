# Next.js学習システム提案書

## 📋 概要

このドキュメントは、**Next.jsで学ぶことのチェックリスト**と**毎日AIが課題を出す仕組み**の設計提案です。

### 目的
- サイト運営を通じてNext.jsを体系的に学習する
- 毎日の作業（5分のNext.js小課題）を自動化・体系化する
- Issue管理と学習進捗を連携させる

---

## 🎯 システムの全体像

```
┌─────────────────────────────────────────┐
│  1. Next.js学習チェックリスト           │
│     （段階的な学習項目を定義）          │
└──────────────┬──────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────┐

└─────────────────────────────────────────┘
```

## 📚 Part 1: Next.js学習チェックリスト

### チェックリストの構造

**ファイル**: `docs/checklists/nextjs-learning-checklist.md`

#### カテゴリ分け（段階的学習）

1. **基礎編（初級）**
   - Next.jsの基本概念
   - ファイル構造の理解
   - 基本的なページ作成

2. **実装編（中級）**
   - コンポーネント設計
   - データフェッチング

3. **最適化編（上級）**
   - パフォーマンス最適化
   - SEO対策
   - デプロイと運用

### チェックリスト項目例

#### 基礎編（初級）

- [ ] **App Routerの理解**
  - [ ] `app/`ディレクトリ構造を理解する
  - [ ] `page.tsx`の役割を理解する
  - [ ] `layout.tsx`の役割を理解する
  - [ ] ファイルベースルーティングを理解する

  - [ ] 静的なページを作成できる

  - [ ] リンクを設定できる（next/link）

- [ ] **スタイリング**
  - [ ] CSS Modulesを使える
  - [ ] グローバルCSSを設定できる
  - [ ] レスポンシブデザインを実装できる

#### 実装編（中級）

- [ ] **コンポーネント設計**
  - [ ] 再利用可能なコンポーネントを作成できる
  - [ ] Propsの型定義ができる（TypeScript）
  - [ ] クライアントコンポーネントとサーバーコンポーネントを理解する
  - [ ] カスタムフックを作成できる

- [ ] **データフェッチング**
  - [ ] `fetch` APIを使える
  - [ ] サーバーコンポーネントでデータを取得できる
  - [ ] クライアントサイドでデータを取得できる
  - [ ] ローディング状態を扱える

- [ ] **ルーティング**
  - [ ] 動的ルート（`[slug]`）を作成できる
  - [ ] クエリパラメータを扱える
  - [ ] リダイレクトを設定できる
  - [ ] 404ページをカスタマイズできる

- [ ] **フォーム処理**
  - [ ] フォームを作成できる
  - [ ] バリデーションを実装できる
  - [ ] サーバーアクションを使える

#### 最適化編（上級）

- [ ] **パフォーマンス最適化**
  - [ ] 画像最適化を理解する
  - [ ] コード分割を理解する
  - [ ] キャッシュ戦略を理解する
  - [ ] バンドルサイズを最適化できる

- [ ] **SEO対策**
  - [ ] メタデータを最適化できる
  - [ ] 構造化データ（JSON-LD）を実装できる
  - [ ] sitemap.xmlを生成できる
  - [ ] robots.txtを設定できる

- [ ] **デプロイと運用**
  - [ ] Vercelにデプロイできる
  - [ ] 環境変数を設定できる
  - [ ] ビルドエラーを解決できる
  - [ ] 本番環境での動作確認ができる

### チェックリストの記録形式

各項目には以下を記録：
- **習得状況**: `未着手` / `学習中` / `完了`
- **実装日**: YYYY-MM-DD形式
- **関連Issue**: Issue番号（例：#123）
- **実装したページ/機能**: 具体的なファイルパスや機能名

## 🤖 Part 3: AI課題生成システム

### システムの動作フロー

```
1. AIが以下を読み込む
   ├─ docs/checklists/nextjs-learning-checklist.md（学習項目）
   ├─ docs/learning-progress/nextjs-progress.md（進捗状況）
   └─ GitHub Issues（label: next の課題）

2. AIが分析
   ├─ 未完了の学習項目を特定
   ├─ 現在のIssue（label: next）を確認
   └─ サイトの改善ニーズを把握

3. AIが課題を提案
   ├─ 学習項目とIssueを組み合わせた課題
   ├─ 難易度（5分で完了できるレベル）
   └─ 実装すべきファイル/機能を明示

4. ユーザーが実行
   ├─ 課題を実装
   ├─ Issueを作成/更新
   └─ 進捗管理ファイルを更新
```

### AI課題生成のルール

#### 課題選定基準

1. **学習項目との連携**
   - 未完了の学習項目から優先的に選ぶ
   - 現在の進捗状況に合わせた難易度

2. **Issueとの連携**
   - `label: next`のIssueがある場合、それを優先
   - Issueの内容を学習項目と組み合わせる

3. **サイト改善との連携**
   - 実装した課題がサイトの改善に直結する
   - 例：「スタイリング」を学びながら「既存ページのレスポンシブ改善」を実装

#### 課題提案の形式

```markdown
## 📝 今日のNext.js課題（5分）

### 学習項目
- [ ] スタイリング > レスポンシブデザインを実装できる

### 実装内容
- `/app/terms/api/page.tsx`のモバイル表示を改善する
- ブレークポイント（768px）でレイアウトを調整する

### 関連Issue
- #55: モバイル表示の改善

### 実装手順
1. `app/terms/api/page.tsx`を開く
2. メディアクエリを追加（`@media (max-width: 768px)`）
3. モバイル表示を確認（開発サーバーで確認）

### 完了条件
- [ ] モバイル表示が適切に表示される
- [ ] ビルドエラーがない
- [ ] Issue #55を更新（実装完了を記録）
```

### AI課題生成のトリガー

#### 手動トリガー
- ユーザーが「今日のNext.js課題を出して」と指示

#### 自動トリガー（将来的な拡張）
- GitHub Actionsで毎日自動実行
- 進捗管理ファイルを自動更新

## 🔄 Part 4: Issue管理との連携

### Issue作成ルール（拡張）

#### 既存のLabel運用
- `next`: Next.js実装タスク（コード変更が必要）

#### 新規Label追加（検討）
- `next-basic`: 基礎編の学習課題
- `next-intermediate`: 実装編の学習課題
- `next-advanced`: 最適化編の学習課題

#### Issueテンプレート（拡張）

**ファイル**: `.github/ISSUE_TEMPLATE/nextjs-learning.md`

```markdown
# Next.js学習課題

## 学習項目
- [ ] （チェックリストから該当項目を記載）

## 実装内容
- （具体的に何を実装するか）

## 関連ファイル
- （実装するファイルのパス）

## 完了条件
- [ ] （完了の基準を明記）

## 学習メモ（任意）
- （学んだこと、気づきを記録）
```

### Issueと進捗管理の同期

1. **Issue作成時**
   - 進捗管理ファイルに`学習中`として記録
   - Issue番号を記録

2. **Issue完了時**
   - 進捗管理ファイルを`完了`に更新
   - 完了日を記録
   - 実装したファイル/機能を記録

## 🎯 運用フロー（1日の流れ）

### 朝（5分のNext.js課題）

1. **AIに指示**
   ```
   「今日のNext.js課題を出して」
   ```

2. **AIが課題を提案**
   - 学習項目を特定
   - Issueを確認
   - 具体的な実装内容を提案

3. **ユーザーが実装**
   - 課題を実装（5分）
   - Issueを作成/更新
   - 進捗管理ファイルを更新

### 週末（振り返り）

1. **進捗確認**
   - 今週完了した学習項目を確認
   - 来週の学習項目を決定

2. **Issue整理**
   - 完了したIssueをClose
   - 未完了のIssueを確認

## ❓ 検討事項

### 1. 学習項目の粒度
- **質問**: 各学習項目はどの程度の粒度が適切か？
- **提案**: 5分で完了できる単位に分割

### 2. 進捗管理の更新頻度
- **質問**: 進捗管理ファイルはいつ更新するか？
- **提案**: 課題完了時に即座に更新（AIが提案時に更新も可能）

### 3. Issueとの連携方法
- **質問**: Issueを自動で作成するか、手動で作成するか？
- **提案**: 最初は手動、慣れてきたら自動化を検討

### 4. 学習項目の追加
- **質問**: 新しい学習項目はどう追加するか？
- **提案**: チェックリストに追加し、進捗管理ファイルにも反映
