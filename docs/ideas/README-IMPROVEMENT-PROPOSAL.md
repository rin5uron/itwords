# README 改善案

**目的**: シンプルに、他人にも自分にもわかりやすく。丁寧に育てる土台であることが伝わる README。

---

## 現状の課題

| 課題 | 具体例 |
|------|--------|
| 情報量が多い | 351行、一度に読むには長い |
| 階層がフラット | 重要度の差がつきにくい |
| 感情が伝わりにくい | ルール中心で「大切にしている」が弱い |
| 振り返りにくい | 自分用の管理ルールが埋もれている |

---

## 改善の方向性

1. **3秒で伝わる** … 最初の数行で「何のサイトか」「どう管理しているか」が分かる
2. **必要に応じて深掘り** … 詳細は docs へのリンクに寄せる
3. **感情を冒頭に** … 「丁寧に育てる」が最初に伝わる
4. **自分用の羅針盤** … 振り返り時に迷わない構成

## このリポジトリについて

このサイトは、門外漢だった自分がITの楽しさを伝えるために作ったものです。
「諦めないでほしい」と届ける想いを、検索で見つけてもらえる形で届けています。

リポジトリは「1ページ = 1Issue」「テンプレート準拠」「データで改善」の3本柱で管理しています。
詳細は docs に集約し、README は羅針盤として最小限に保ちます。
