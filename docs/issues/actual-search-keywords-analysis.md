# 実際の検索キーワード調査・分析

**作成日**: 2026年1月24日  
**目的**: ユーザーが実際にどのようなキーワードで検索しているかを調査し、SEO戦略に反映する

---

## 🔍 調査の背景

ユーザーが「ガガ」「ライブ」「予習」などで調べたとのこと。これは実際の検索行動パターンを理解する重要なヒントです。

### 想定される検索パターン

1. **「ガガ」で検索**
   - 可能性: 「API」を「エーピーアイ」と読めずに「ガガ」と検索？
   - または: 別のIT用語の読み間違い？
   - 対策: 読み方のバリエーションを考慮したキーワード戦略

2. **「ライブ」で検索**
   - 可能性: 「ライブ」という言葉でIT用語を調べた？
   - または: 「ライブコーディング」「ライブリロード」などの関連用語？
   - 対策: 関連用語の検索意図を分析

3. **「予習」で検索**
   - 可能性: 「IT用語 予習」「プログラミング 予習」などの検索意図
   - 対策: 学習意図を含むキーワード戦略

---

## 📊 実際の検索キーワード調査方法

### 1. Google Search Consoleの確認（最優先）

**確認すべきデータ**:
- 実際にサイトに来ている検索キーワード
- クリック率（CTR）が低いキーワード
- 表示回数は多いがクリックされていないキーワード
- 順位11-20位の「惜しい」キーワード

**アクション**:
1. Search Consoleにログイン
2. 「パフォーマンス」→「検索結果」を確認
3. 上位20キーワードをリスト化
4. 各キーワードのCTR、順位、表示回数を記録

---

### 2. Google検索サジェストの確認

**方法**:
1. Google検索ボックスに「IT用語」と入力
2. 表示されるサジェストを記録
3. 各用語ページのキーワードでも同様に確認

**確認すべきキーワード**:
- `IT用語`
- `IT用語 とは`
- `IT用語 初心者`
- `IT用語 わかりやすく`
- `API とは`
- `JSON とは`
- `フールプルーフ とは`
- など、各用語ページのメインキーワード

---

### 3. Google Trendsでの検索トレンド確認

**確認すべき内容**:
- 各IT用語の検索トレンド（過去1年、5年）
- 関連キーワードの検索ボリューム
- 季節性（特定の時期に検索が増えるか）

**アクセス**: https://trends.google.co.jp/

---

### 4. 競合サイトのキーワード分析

**確認すべきサイト**:
- IT用語辞典 e-Words
- カオナビ IT用語辞典
- その他上位表示されているサイト

**確認方法**:
- 各サイトのタイトルタグ、メタディスクリプションを確認
- どのキーワードで上位表示されているか分析

## 🎯 実際の検索パターン分析（仮説）

### パターン1: 読み方のバリエーション

**例**:
- 「API」→「エーピーアイ」「ガガ」（読み間違い）
- 「JSON」→「ジェイソン」「ジョン」
- 「DNS」→「ディーエヌエス」「ドメイン」

**対策**:
- ページ内に「読み方」を明確に記載
- メタキーワードに読み方のバリエーションを含める
- FAQに「読み方は？」を追加

---

### パターン2: 学習意図を含む検索

**例**:
- 「IT用語 予習」
- 「プログラミング 予習」
- 「IT用語 勉強」
- 「IT用語 覚え方」

**対策**:
- 「学習ガイド」セクションを追加
- 「初心者向け学習順序」ページを作成
- 各ページに「次のステップ」セクションを追加

---

### パターン3: 実践的な検索

**例**:
- 「API 使い方」
- 「JSON 編集」
- 「LocalStorage 保存」
- 「Webhook 送信」

**対策**:
- デモを前面に出す
- 「実践的な使い方」セクションを強化
- 「よくある使い方」を追加

---

### パターン4: 比較・違いを調べる検索

**例**:
- 「API REST 違い」
- 「JSON XML 違い」
- 「LocalStorage Cookie 違い」
- 「フールプルーフ ポカヨケ 違い」

**対策**:
- 比較表を追加
- 「関連用語との違い」セクションを強化
- 内部リンクで関連ページに誘導
