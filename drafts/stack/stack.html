スタック構造 (Stack)

よみかた：すたっくこうぞう

意味

最後に入れたものが、最初に出てくるという仕組みのデータ構造のこと。
専門用語で「LIFO（Last In, First Out）」と呼ばれます。
積み重ねた本やお皿の山を想像するとわかりやすいでしょう。

ポイント

  • 「上に積む (push)」「上から取る (pop)」が基本操作です。
  • ウェブブラウザの「戻る」ボタン、アプリケーションの「元に戻す(Undo)」機能、プログラミングの関数呼び出し履歴など、身近なところで使われています。
  • 順番どおりに処理したいときに便利ですが、途中のデータ（真ん中のお皿など）を直接取り出すのは苦手です。

実際に体験してみよう

下のボタンを押して、スタックの「後入れ先出し」を体験してみましょう。「Push」でデータを追加し、「Pop」でデータを取り出せます。

関連用語・比較

用語名 構造 特徴
スタック 後入れ先出し (LIFO) 「戻る」操作や、一時的な処理に強い
キュー 先入れ先出し (FIFO) 受付や行列のような順番通りの処理に向く
配列 (Array) 任意の位置にアクセス可能 特定のデータを直接取り出せる

コードで見てみよう

git stash実践シナリオ

バージョン管理ツールGitの一次保存コマンド git stash は、スタック構造の代表的な応用例です。変更内容を一時的に「積み重ねて」退避させることができます。

サンプルコードを見る (Bash) 
# 作業内容をスタックに積む
$ git stash

# 最後に積んだものを取り出して適用する
$ git stash pop

# スタックの中身を一覧で確認する
$ git stash list

# 特定のスタックを取り出して適用する (例: 1番目)
$ git stash apply stash@{1}

🧠 覚えておきたいポイント：
stash するたびに変更が「上に積まれ」、pop で「一番上＝最後に積んだもの」が取り出されます。まさにスタックそのものです。

スタック構造とは、「積み重ねて、上から取り出す」ルールのこと。
身近な「戻る」機能や、中断した作業を管理する場面などで活躍します！
